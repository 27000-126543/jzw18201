## 1. 产品概述

酒店客房在住服务管理系统——专注于提升客人入住期间的服务体验。客人通过扫描房间二维码即可提交各类服务请求（叫餐、补备品、报修、叫醒、免打扰），请求自动分发至对应部门工单池，各部门操作人员接单处理后更新状态，客人可实时追踪进度。前台大屏展示待处理请求与响应时长，服务记录关联客户档案供下次入住参考，退房前推送满意度评价，差评实时通知当班经理。

- 目标用户：酒店入住客人、前台接待、客房部/餐饮部/工程部操作人员、当班经理
- 核心价值：缩短服务响应时间、提升入住体验、数据驱动服务改进

## 2. 核心功能

### 2.1 用户角色

| 角色 | 进入方式 | 核心权限 |
|------|----------|----------|
| 入住客人 | 扫描房间二维码 | 提交服务请求、查看进度、满意度评价 |
| 前台接待 | 后台账号登录 | 查看工单池、分配工单、查看大屏 |
| 客房部人员 | 后台账号登录 | 接收/处理备品补充与免打扰类工单 |
| 餐饮部人员 | 后台账号登录 | 接收/处理叫餐类工单 |
| 工程部人员 | 后台账号登录 | 接收/处理报修类工单 |
| 当班经理 | 后台账号登录 | 全局监控、差评实时推送、查看报表 |

### 2.2 功能模块

1. **客人服务页**：扫码入口、服务类型选择（叫餐/备品/报修/叫醒/免打扰）、在线点餐菜单、请求提交与进度追踪、满意度评价
2. **后台工单管理页**：工单池总览、按部门筛选、工单状态流转、响应时长统计
3. **前台大屏页**：待处理请求实时看板、响应时长排行、部门负载概览
4. **客户档案页**：入住服务历史、偏好标签、满意度趋势

## 4. 用户界面设计

### 4.1 设计风格

- **主色调**：深金色(#C9A96E) + 暖灰白(#FAF8F5)，传达高端酒店质感
- **辅助色**：深炭灰(#2C2C2C)用于文字与深色区块，柔白(#FFFFFF)用于卡片底色
- **强调色**：珊瑚橙(#E8734A)用于紧急/待处理标识，翠绿(#4CAF7D)用于完成状态
- **按钮风格**：圆角8px，主按钮深金色填充，次按钮描边，hover时微上浮+阴影加深
- **字体**：标题使用 Playfair Display 衬线体（高端质感），正文使用 Noto Sans SC（中文友好）
- **布局风格**：卡片式布局为主，左侧导航栏，内容区滚动
- **图标风格**：线性描边图标(24px)，统一2px线宽，圆角端点

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI元素 |
|----------|----------|--------|
| 客人服务页 | 扫码入口 | 全屏欢迎页，酒店Logo居中，房间号大字显示，五服务类型圆角卡片网格，金色渐变顶栏 |
| 客人服务页 | 在线点餐 | 左侧分类导航，右侧菜品卡片(图片+名称+价格)，底部悬浮购物车，提交按钮金色 |
| 客人服务页 | 我的请求 | 请求卡片列表，每卡显示类型图标+标题+状态标签+进度条，下拉刷新 |
| 客人服务页 | 满意度评价 | 全屏弹窗，星级交互(1-5星)，分项滑块评价，文字输入框，提交按钮 |
| 后台工单管理页 | 工单池 | 顶部筛选栏(状态/部门/类型)，工单卡片列表(编号+类型+房间+时长+状态徽章) |
| 后台工单管理页 | 工单详情 | 侧滑抽屉面板，工单信息+操作按钮+状态时间线 |
| 前台大屏页 | 实时看板 | 深色主题，顶部汇总数字大字，左侧待处理列表滚动，右侧部门负载环形图，响应时长条形图 |
| 客户档案页 | 服务历史 | 时间轴布局，入住记录卡片，展开显示服务明细与评价 |
| 客户档案页 | 偏好标签 | 标签云展示，金色/灰色标签区分高频/低频偏好 |

### 4.3 响应式设计

- 桌面优先设计，客人服务页同时适配移动端（扫码后多为手机使用）
- 客人端：移动端优先，375px起适配，卡片单列/双列自适应
- 后台与大屏：桌面端优先，1280px+最佳体验，1024px以上可用
- 触摸优化：客人端按钮最小44px触摸区域，滑动操作支持
